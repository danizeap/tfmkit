# Documentation Standards

## Source Of Truth

- `PROJECT_CONTEXT.md` explains what the project is and why it exists.
- `README.md` explains how to use or run the project.
- `sdd-plus/standards/` defines how work should be done.
- `sdd-plus/specs/` stores durable product and technical specs.
- `sdd-plus/changes/` stores active change packets.
- `sdd-plus/archive/` stores completed change packets.

## Required Updates

Update documentation when a change affects:

- setup or run commands
- environment variables
- data models or schemas
- API contracts
- authentication, authorization, or security assumptions
- deployment process
- user-facing behavior
- model/AI usage
- important operating decisions

## Style

- Be concrete.
- Prefer commands and exact file paths.
- Mark assumptions and open questions.
- Keep durable facts in files, not only in chat.
